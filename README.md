# Tic-Tac-Toe Game

Веб-приложение для игры в крестики-нолики с поддержкой игры против ИИ и между двумя игроками. Реализована система авторизации, валидация ходов и сохранение состояния игры в базе данных.

## Описание

Tic-Tac-Toe Game — это полнофункциональное веб-приложение, позволяющее играть в крестики-нолики в двух режимах:
- **Игра против компьютера** — с использованием алгоритма Minimax для оптимальных ходов ИИ
- **Игра между двумя игроками** — поддержка многопользовательского режима

Приложение включает систему регистрации и авторизации пользователей, валидацию игровых ходов для предотвращения читерства, и автоматическое сохранение состояния игры.

## Технологии

- **Backend:**
  - C# (.NET 8.0)
  - ASP.NET Core 8.0 (Razor Pages + REST API)
  - Entity Framework Core 8.0
  - PostgreSQL (с JSONB для хранения игровых данных)
  
- **Frontend:**
  - Razor Pages
  - JavaScript (ES6+)
  - Bootstrap 5
  
- **Архитектура:**
  - Многослойная архитектура (Domain, DataSource, Web)
  - Repository Pattern
  - Dependency Injection
  - Mapper Pattern

## Основные возможности

- Регистрация и авторизация пользователей (Basic Authentication)
- Игра против компьютера с алгоритмом Minimax
- Игра между двумя игроками
- Валидация игровых ходов
- Сохранение истории ходов
- Автоматическая инициализация базы данных
- RESTful API для работы с играми

## Архитектура проекта

Проект организован по принципу многослойной архитектуры:

```
Tic-Tac-Toe/
├── domain/              # Доменный слой (бизнес-логика)
│   ├── model/          # Доменные модели (Game, User, GameBoard)
│   └── service/        # Бизнес-логика (GameService, IGameService)
│
├── datasource/          # Слой доступа к данным
│   ├── dbcontext/      # EF Core контекст и конвертеры
│   ├── model/          # DTO модели для БД
│   ├── mapper/         # Маппинг Domain ↔ DTO
│   ├── repository/     # Репозитории для работы с БД
│   └── service/        # Адаптеры для доменных сервисов
│
├── web/                 # Веб-слой
│   ├── controller/     # API контроллеры
│   ├── middleware/     # Middleware и фильтры
│   ├── model/          # Модели запросов/ответов
│   ├── mapper/         # Маппинг Domain ↔ Web модели
│   └── service/        # Веб-сервисы (AuthService)
│
├── di/                  # Конфигурация Dependency Injection
├── Pages/               # Razor Pages (UI)
└── wwwroot/             # Статические файлы (CSS, JS)
```

## API Endpoints

### Авторизация
- `POST /auth/register` — Регистрация нового пользователя
- `POST /auth/login` — Вход в систему

### Игры
- `GET /game/{id}` — Получить игру по ID
- `POST /game` — Создать новую игру
- `POST /game/{id}` — Сделать ход в игре

Все API endpoints (кроме `/auth/register` и `/auth/login`) требуют Basic Authentication.

## Ключевые особенности реализации

### 1. Многослойная архитектура
Проект разделен на три основных слоя (Domain, DataSource, Web) с четким разделением ответственности, что обеспечивает тестируемость и поддерживаемость кода.

### 2. Алгоритм Minimax
Реализован AI-противник на алгоритме Minimax для поиска оптимальных ходов компьютера в игре против человека.

### 3. Работа с PostgreSQL и JSONB
Настроена работа с PostgreSQL через EF Core:
- Автоматическая инициализация БД при старте приложения
- Миграции для управления схемой БД
- Кастомные конвертеры данных для преобразования сложных типов (двумерные массивы, списки) в формат JSONB PostgreSQL

### 4. Система авторизации
Реализована система авторизации через Basic Authentication с глобальным фильтром авторизации, применяемым ко всем API-эндпоинтам автоматически, и валидация игровых ходов для предотвращения изменения предыдущих ходов.
